# Song Anatomy — 「この曲を解剖して」

楽曲ファイルまたは YouTube URL を受け取り、全分析ツールを順番に呼び出して、
DJ/音楽制作の先生として「なぜこの曲が気持ちいいか」を平易な日本語で解説する。

## 手順

### Step 1: 音声読み込み
`load_audio` を呼び出す。ユーザーがファイルパスまたは YouTube URL を指定している場合はそれを使う。

### Step 2: 基礎分析（並列）
以下の3つを並列で呼び出す:
- `analyze_bpm` → BPM
- `analyze_key` → キー（調）
- `analyze_energy` → エネルギーカーブ画像

### Step 3: 構造認識
- `analyze_structure` → セクションリスト（Intro/Build/Drop/Break/Outro）

### Step 4: 可視化（並列）
以下の2つを並列で呼び出す:
- `visualize_spectrogram` → スペクトログラム画像
- `visualize_waveform` → 波形 + セクションアノテーション画像
  - Step 3 で得た sections を渡す

### Step 5: 解説
すべての分析結果を踏まえて、以下の構成で日本語で解説する:

#### 基本情報
- BPM、キー、長さを簡潔に

#### 構造マップ
- セクション一覧を時系列で表示（時刻 + ラベル + エネルギー）
- 例: `0:00-0:28 Intro（静か、期待感を作る）`

#### なぜこの曲が気持ちいいか
以下の観点から語る:
1. **エネルギーの流れ**: エネルギーカーブの起伏から、テンション→リリースの構造を読み解く
2. **キーとスケール**: メジャー/マイナーの感情的な効果、DJ的な視点での相性
3. **BPMとグルーヴ**: ジャンル的な文脈（UKG=130前後、Deep House=120前後 etc.）
4. **構造の巧みさ**: Intro→Build→Dropの流れ、Break の使い方、Outro のミックスしやすさ

#### DJへのヒント
- この曲の前後に合いそうなBPM/キーの範囲
- ミックスポイントの提案（Intro/Outro の長さから）
- セットの中での役割（オープナー？ピーク？クローザー？）

## トーン
- 音楽理論の初心者にもわかる平易な日本語
- 専門用語を使う場合は必ず（）で簡単な説明を添える
- 「先生が生徒に教える」のではなく「音楽好きの友達が語る」テンション
- 画像は必ずユーザーに見せる（分析結果の裏付けとして）
